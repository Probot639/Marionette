# protocol

Wire format and message catalog for doll-to-teamserver traffic.

## Status

`frame.go` (envelope, framing, AEAD wrap), `handshake.go` (Register, RegisterAck, Config), and `beacon.go` (Beacon, BeaconAck, Result, String, Status) are wired up and tested. `strings.go` (the v1 String type catalog with Shell/Sleep/Exit/Cd/Pwd/Whoami payload encoders) is next.

## Wire shape

Two layers. Outer is what hits the wire (and what listeners forward without
parsing). Inner is what you get after AEAD-opening.

```
plaintext message (Register, RegisterAck):

   0    1    2                              n
   +----+----+------------------------------+
   | ver| typ|             body             |
   +----+----+------------------------------+

encrypted message (Beacon, BeaconAck):

   0          12                             n
   +-----------+----------------------------+
   |   nonce   |       ciphertext           |
   +-----------+----------------------------+

   ciphertext after AEAD-open:

   0    1    2                              n
   +----+----+------------------------------+
   | ver| typ|             body             |
   +----+----+------------------------------+
```

`ver` is the protocol version (currently `1`). `typ` is the MessageType byte
(see `frame.go`). `nonce` is a fresh 12-byte AES-GCM nonce. `body` is
type-specific.

The version and type bytes live inside the encrypted envelope by design, so a
passive listener can not tell a Beacon from a BeaconAck without the key.

## Messages

### Register (doll to server, plaintext)

Sent on first contact. No session key exists yet, so the message is plaintext
on the wire.

```
body: [doll_uuid:16][doll_pubkey:32]
```

`doll_uuid` is a UUID generated by the doll at first run. `doll_pubkey` is the
doll's ephemeral X25519 pubkey for the session handshake.

### RegisterAck (server to doll, plaintext outer + sealed config)

Outer frame is plaintext because the doll does not have the session key yet.
Body holds the server's ephemeral pubkey in the clear, followed by an
AEAD-sealed initial config.

```
body: [server_pubkey:32][nonce:12][sealed_config:bytes]
```

`sealed_config` after AEAD-open with the just-derived session key:

```
[sleep_ms:varint][jitter_pct:u8]
```

Doll flow:

1. Read `server_pubkey` from the first 32 bytes of the body.
2. Derive the session key with `crypto.DeriveSharedKey(doll_priv, server_pubkey)`.
3. Open the rest with that key to recover the initial config.

### Beacon (doll to server, encrypted)

Doll polling for strings. May carry results from previously-issued strings.

```
body: [num_results:varint][results...]

each result: [string_id:u64][status:u8][body_len:varint][body:bytes]
```

### BeaconAck (server to doll, encrypted)

Server's reply to Beacon. Carries any pending strings.

```
body: [num_strings:varint][strings...]

each string: [string_id:u64][string_type:u8][payload_len:varint][payload:bytes]
```

## Status codes

```
0  OK
1  Error
2  Timeout
```

## String types (v1)

| Byte | Name   | Payload              |
|------|--------|----------------------|
| 1    | Shell  | command bytes        |
| 2    | Sleep  | new sleep_ms varint  |
| 3    | Exit   | empty                |
| 4    | Cd     | target path bytes    |
| 5    | Pwd    | empty                |
| 6    | Whoami | empty                |

## Conventions

- Fixed-width integers are little-endian.
- All length prefixes are unsigned varints (`encoding/binary.PutUvarint`).
- Max body size is 1 MB for v1. Chunking comes when whisper lands.
- AAD is unused for v1. Likely candidate later: doll UUID plus a session counter.

## Adding new types

1. Add the constant to `frame.go` (MessageType) or `strings.go` (StringType).
2. Add the body marshaler in the appropriate file.
3. Add a round-trip test.
4. Bump the protocol version byte if the change is wire-incompatible.
